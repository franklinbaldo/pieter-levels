# memory/notes/monetization-strategy.md

## Core Strategy (Pieter Levels approach)
1. **Leverage what exists:** Do not suggest building new massive tools. Find value in existing data (`causaganha`, `baliza`), APIs (`verne`), and processes (`egregora`).
2. **Simple wrappers:** Package existing functionality into single-purpose, simple SaaS products or APIs.
3. **Validate before building:** Use landing pages (Carrd, Astro) to test willingness to pay via simple Stripe payment links before writing any billing code.
4. **Target Audiences:** Law firms (`causaganha`), businesses bidding on gov contracts (`baliza`), founders/creators (`egregora`), indie hackers (`verne`).

## Daily Scans
- The daily scan (`reports/latest.md`) aims to track new commits, deployments, and PRs across Franklin's projects to identify *new* monetization signals.
- So far, Franklin's focus is on OSS and exploration. The job is to identify the pivot point from "cool tech" to "paid product."
